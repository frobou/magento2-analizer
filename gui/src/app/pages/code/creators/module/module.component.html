O que é um módulo?

É um lugar onde podemos adicionar funcionalidades ou alterar comportamentos existentes

Existem alguns tipos possíveis (tema, módulo, internacionalização), aqui vamos ver a estrutura básica de um módulo

Os tipos de módulos

Criando um módulo

A primeira etapa é definir o vendor e o nome

Vendor é o fornecedor, pode ser o nome da empresa, por exemplo

Já o nome deve ser algo que indique o motivo da sua criação, customer, por exemplo

Agora que definimos um nome, seguimos para a próxima fase

Mão na massa

Existe uma estrutura e alguns arquivos que são obrigatórios, além de poder ser criado de algumas foramas diferentes, vamos ver a mais simples aqui

app = pasta padrão do magento onde ficam diversos arquivos e pastas

code = pasta onde os módulos devem ser colocados (se naão existir, devemos criar)

Vendor = pasta do fornecedor do módulo, podemos ter vários módulos por fornecedor (Vendor é um exemplo)

Customer = pasta onde os arquivos do módulo ficam (Customer é um exemplo)

Tanto a pasta do vendor quanto a pasta do módulo devem iniciar com letra MAIÚSCULA (podendo usar também o padrão CamelCase)

etc = pasta onde os "arquivos de configuração" do módulo ficam

module.xml = arquivo de definição do módulo
Note que o nome do módulo ficou como "Vendor_Customer", é assim que deve ser, o vendor e o nome, separado por um "_"

registration.php = arquivo que o magento usa para adicionar o módulo ao framework
Assim como no module.xml, o vendor e o nome devem estar separado por um "_"

Existe ainda um arquivo opcional, o composer.json, que pode ser usado se queisermos criar um módulo para ser distribuído/reutilizado
Note que, diferente do module.xml, a chave psr-4 fica como "Vendor\\Customer\\", isso é por causa do autoload do composer

Hora de fazer

Crie os arquivos de acordo com a imagem, eles devem estar em seus exatos lugares...

*Valores em negrito são os nomes de sua escolha, e devem estar em CamelCase
  • vendor em app/code
  • name em app/code/vendor
  • registration.php em app/code/vendor/name
  • composer.json em app/code/vendor/name (opcional)
  • etc em app/code/vendor/name
  • module.xml em app/code/vendor/name/etc
